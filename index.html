HTML

  • HTML: Hypertext Markup Language: is what gives the content it's structure.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS: Cascading Style Sheets: gives the content style.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Marigin and padding are often confused. Padding is trnsparent but makes the element bigger, while the margin is the transparent buffer around the element.
  • See notebook for "CSS chapter 3.2.3"

Git

  • Git is an open source distributed version control system for everything GitHub related that happens locally on a computer.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • ls: a list of files and directories on the computer
  • cd: allows you to change directions. ie "cd Downloads" takes you on the Download file
  • pwd: stands for print working directory, shows you where/how to get to the selected file
  • mkdir "name of directory": creates a directory
  • touch "new file name": creates a file in a directory. ie "first-day.txt"
  • start "first-day.txt": opens the file in a directory

JavaScript

  • JS: JavaScript: handles logic and allows user to interact with a website.
  • JS: high-level dynamic programming language that allows users to interact with websites
  • Powers your websites with effects, interactivity, data communication, and much more.
  • "Control flow is the order in which a computer executes code in a script."
  • "In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page."
